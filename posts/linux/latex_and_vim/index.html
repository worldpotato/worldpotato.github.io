Using Vim for LaTeX together with zathura

2020-05-01

I used LaTeX for the last two semesters to write my scripts for different lessons at university. Mostly because it is based on plain text files and that makes it perfect for git.

Most of the time I used TexStudio. It’s a great TeX editor. But I like vim and thought that I should figure out how to get a nice workflow with vim. And of course I did some ducking and found the latex-suite. For some reasons it doesn’t worked well. There were some problems with compiling, which I could not fix. Because I tried it while I heard a more or less boring lesson, I decided to stay with TexStudio.

That was around six month ago. Yesterday evening I decided to give it a shot again. But that time I decided to use an other plugin for TeX. I tried LaTeX-Box and liked how it works with YouCompleteMe and Syntastic. I was happy for the day and worked a little bit on my bachelor thesis. After some adaptions the autocompletion worked well, and the syntax highlighting looks good and I get hint if my syntax is not correct.

Today I thought that I miss the forward and backward search from TexStudio. So I took a quick look what I can find on duckduckgo.com how to get the forward and backward search. And that’s how I found vimtex which is based on LaTeX-Box. To be honest, it doesn’t worked out of the box. I had to do some small adaptions to my vimrc.

Let’s talk about that.

tex flavor

The first problem was that vim handles files with *.tex ending as plaintext files. But to have a nice workflow vim should handle these files as what they are.

Adding this line helps here.

let g:tex_flavor='latex'

Fixing forward/backward search

I set zathura as my standard pdf viewer so it opens with the <leader>lv combination from vimtex. But it doesn’t work with the backward search. And that had two reasons.

  1. vimtex used xdg-open to open the pdf viewer
  2. vim’s servername string is empty

I found these two thing out while reading through this GitHub issue.

To each of these problems is a simple solution.

  1. Set zathura explicit as your view method by adding this line to your vimrc
let g:vimtex_view_method = 'zathura'
  1. Set the servername when you start vim. Or better, set an alias.
vim --servername vim

For me it works quite good now. And I’m really stroked about how good I can use one editor to edit different filetypes.
