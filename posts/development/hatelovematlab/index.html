My hate/love to Matlab

2020-04-18

I don’t really know where to start. I work with MatLab in university for the last three years, mostly for implementing different algorithm. But the last days I was working on a little MatLab app. And that experience was not one of the nice ones. I thought I should share that.

Let me tell, at some point I really hate MatLab and some seconds later I love MatLab again. And to be honest, overall I don’t really enjoy developing software with Matlab. But I love writing small prototypes of algorithm and solving mathematical problems with it. And that is really nice. It works fast, easy and uncomplicated.

Until you need to use bigger toolboxes. That’s the moment where you should ask yourself if MatLab is the right tool. Because MatLab is not made to develop bigger applications with it. And you can see it on different things. For example, multitasking is not really possible. Even if you have the Parallel Toolbox installed it still feels clumsy. Or talking about objects, working with objects getting from functions is nice. But creating own objects and using these is exhausting compared to java or python.

An other thing is the very closed environment. Actually it’s OK, but also far away from being great. Types are not recognized during writing code, tab completion is there but it is really not good. And a lot of things are saved in a proprietary binary format.

There is also the documentation. I heard a lot of good things about it, and indeed there is a documentation about every function. But you can not find anything about the input and output types of the functions. Which leads you to try it in the MatLab console before using it in your script.

I know these bad things are not really good discussed from my side but I just want to talk about it. And I want to say that the simple things you can do in MatLab are really nice. So choose your tool depending on what you want to do. Otherwise you will choose a hammer for a screw. It works, but does it work good?
